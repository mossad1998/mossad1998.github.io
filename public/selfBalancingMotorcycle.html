OS for Self-Balancing Motorcycle

This project was my senior graduation project in the Mechatronics and Automation Department at Ain Shams University. It represents a comprehensive study and implementation of the control system, embedded electronics, and dynamics behind a two-wheel self-balancing electric motorcycle, stabilized using gyroscopic effects. The vehicle was designed to serve as a bridge between cars and motorcycles â€” offering single-seat efficiency, zero emissions, and cost-effectiveness compared to traditional fuel-powered vehicles.

Core Balancing Concept

The motorcycle uses a Single Gimbal Control Moment Gyroscope (SGCMG) as the main actuator for stabilization. Unlike reaction wheels, SGCMG provides higher torque amplification and momentum exchange capacity, making it suitable for counteracting unexpected inclinations. By controlling both gimbal speed and gimbal angle, the gyroscope generates corrective torques to maintain stability during disturbances.





Embedded Architecture

  • Gyroscope BLDC Motors: Drive the gimbal rotor up to 3000 RPM for momentum generation.
  • ESCs (Electronic Speed Controllers): Control BLDC motor velocity with closed-loop feedback.
  • Encoders: Measure real-time rotor speed for precise control.
  • DC Motors + Controllers: Rotate the gimbal to counteract disturbances.
  • Rotary Potentiometers: Measure gimbal angular displacement.
  • IMU (Inertial Measurement Unit): Provides roll/pitch inclination feedback.
  • Embedded Controller: ARM Cortex-M4F (Tiva-C) running FreeRTOS to schedule and manage concurrent tasks (sensor fusion, gimbal speed control, gimbal angle control, fault handling).
  • Fuse Protection: Protects system electronics against overcurrent scenarios.

Controlled Parameters

  1. Gimbal Speed Control: Maintains rotor angular velocity at a stable reference point for constant momentum storage.
  2. Gimbal Angle Control: Adjusts gimbal orientation to generate corrective torques against external disturbances.

System Identification & Control Design

We applied system identification techniques to model the dynamic behavior of the gyroscope drive system:

  • Input: Ramp signals applied to BLDC motors.
  • Output: Speed feedback measured from encoders.
  • Processing: Filtering and noise suppression using moving average methods.
  • Modeling: Transfer function estimation using MATLAB System Identification Toolbox.
