Multi-junction Traffic Light Controller

This project demonstrates a prototype for an Intelligent Transportation System (ITS) by implementing a realistic multi-junction traffic light controller. The design integrates road intersections with pedestrian crossings and a railway line, coordinated through real-time embedded software to manage traffic flow, pedestrian safety, and train priority.

The hardware prototype includes:

  • Four traffic light modules (north-south, east-west).
  • Four pedestrian push-buttons linked to each light.
  • Two train push-buttons (approach and departure sensors).
  • Servo-motor controlled railway crossing gate.
  • Audible siren for train approach alerts.
  • Project Link on GitHub

System Behavior

  1. Road Priority: Because the two main roads are perpendicular, one road is active at a time (NS: 5s, EW: 2.5s).
  2. Pedestrian Crossing: A button press while the signal is green interrupts the current phase, halts vehicles, and activates pedestrian crossing mode.
  3. Train Priority: When the train approach button is pressed, all traffic lights flash red at 1 Hz, the siren is activated, and the gate is lowered.
  4. Train Departure: After 20s (minimum crossing time) and confirmation of departure via second push-button, the system resumes normal operation.

Embedded System Design

To meet real-time constraints, the project was implemented using:

  • FreeRTOS Kernel: Used to schedule and manage four concurrent tasks (traffic control, pedestrian service, train handling, and gate control).
  • Binary Semaphores & Interrupts: Switch inputs (pedestrian/train) trigger interrupts, signaling tasks via semaphores â€” enabling responsive, low-power operation.
  • General-Purpose Timers (GPTM):
    • PWM generation for servo motor controlling the railway gate.
    • Timing services for light cycles (20s, 5s, and 2.5s).
  • Deterministic Scheduling: Priority-based task execution ensured train events always preempt road/ pedestrian cycles, reflecting real-world ITS safety requirements.

Test Scenarios (Operation Manual)

  • Reset: System boots into normal cycle mode (default scheduling).
  • Pedestrian request on red: Ignored by the system (no effect).
  • Pedestrian request on green: Vehicle lights transition to red, pedestrian signal activated.
  • Multiple simultaneous requests: Prioritized based on real-time task hierarchy.
  • Train approach: Gate closes, red lights flash, siren engages.
  • Early departure signal: Ignored until crossing time expires.
  • Valid departure signal (post 20s): Gate opens, lights return to normal sequence.
