Telemetry & Data Logger with Closed-Loop System Identification

This project focused on developing a custom telemetry and data logging system for capturing and analyzing real-time signals from embedded control systems. The system is designed to intercept and record data communicated over the UART bus between a microcontroller and its connected sensors or actuators. It logs live telemetry into structured CSV files for post-processing and control system evaluation.

The primary application was built around a prototype steering-by-wire system, which emulates electronically controlled steering using a 12V DC motor. The system was subjected to open-loop step inputs and closed-loop feedback control to characterize its behavior. Acquired data was analyzed using MATLAB’s System Identification Toolbox to estimate a reliable transfer function model of the physical process.

Embedded Design & Features

  • Embedded firmware written in C/C++ running on an STM32 microcontroller using bare-metal programming (HAL drivers) to handle UART interrupts and data parsing.
  • Logged parameters included control input (voltage/PWM), motor angular position, and current feedback at fixed sampling intervals.
  • Implemented ring buffer for efficient UART packet handling and DMA-based logging to minimize CPU overhead.
  • Developed a lightweight host application in Python to parse serial logs and store them in timestamped CSV files.
  • Integrated signal processing filters (moving average and weighted moving average) to eliminate high-frequency sensor noise.

Steering-by-Wire Prototype

To validate the data logger’s functionality, a hardware-in-the-loop steering model was developed using:
  • Brushed DC motor (12V) with encoder feedback.
  • Linear potentiometer for position feedback.
  • Open-loop and closed-loop control via PID algorithm implemented on STM32.
  • Telemetry logging used to compare actual vs. target trajectory and validate dynamic system performance.

System Identification Workflow

1. Apply known step inputs and capture real-time system response using the telemetry logger.
2. Import the dataset into MATLAB and process it using signal conditioning filters.
3. Use the System Identification Toolbox to fit a transfer function or state-space model.
4. Evaluate model fidelity by comparing simulation output to real measured data.
5. Re-run experiments with closed-loop PID control for validation and stability analysis.
